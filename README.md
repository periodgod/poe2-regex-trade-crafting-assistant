# POE 多区服正则、兑换、集市监控与严格做装助手 v1.9.0

本项目是本地运行的 POE 桌面工具包，包含四个工作区：

1. **完整正则生成器**：装备、引路石、石板、仓库与高级条件。
2. **五资源兑换助手**：20 个双向兑换方向、金币手续费、持仓路线和闭环分析。
3. **四区服集市实时监控**：流放 2 国服/国际服、流放 1 国服/国际服。
4. **POE2 严格做装状态机**：精确底材词缀池、通货、骨材、精华、符文、灵魂核心、合金、预兆与亵渎揭示。

## 文档约定

版本更新、Bug 修复和验证结果统一维护在本 README 中。后续更新不再新增 `FIX_SUMMARY_*`、`TEST_REPORT_*` 或独立热修复说明文件。

## Windows 启动

必须先对 ZIP 右键选择 **“全部解压”**，再进入完整目录。优先双击：

```text
START_APP_DIRECT.cmd
```

也可以运行：

```text
START_HERE.cmd
```

所有 `.cmd` 使用 Windows CRLF 且不带 BOM；菜单主体由 UTF-8 BOM PowerShell 脚本执行，兼容中文路径与 Windows PowerShell 5.1。

## v1.9.0 当前更新

### 四区服市场监控

- 市场监控扩展为流放 2 国服、流放 2 国际服、流放 1 国服、流放 1 国际服四区服框架。
- 监控项目保存 `serverKey` 和 `serverLabel`，链接解析、API endpoint 和详情读取按区服分流。
- 页面支持默认区服和混合区服批量链接；每条链接自动生成独立监控项目。
- 通货筛选扩展到 POE1 常见通货，支持每种通货独立价格区间。
- 保存层保留监控名称、通货多选、价格区间和区服信息，重启后不会丢失。

| 区服键 | 游戏 | 地区 | 搜索路径 |
|---|---|---|---|
| `poe2-cn` | 流放之路 2 | 国服 | `/trade2/search/poe2/{league}/{query}` |
| `poe2-intl` | Path of Exile 2 | 国际服 | `/trade2/search/poe2/{league}/{query}` |
| `poe1-cn` | 流放之路 1 | 国服 | `/trade/search/{league}/{query}` |
| `poe1-intl` | Path of Exile 1 | 国际服 | `/trade/search/{league}/{query}` |

### 国服监控与限流

- 支持 `poe.game.qq.com/trade2/search/poe2/...` 国服链接，以及官方国际服 POE1/POE2 搜索链接。
- 国服采用 GET 搜索定义后再 POST 执行搜索，支持商人页签和前往藏身处模式。
- 多个项目进入同一串行队列，国服请求至少间隔约 65 秒。
- 遇到 HTTP 429 时暂停整个队列，按 `Retry-After` 或默认冷却时间恢复。
- Cookie 仅用于当前会话请求，不写入监控项目或本地持久化数据。

### 监控工作流

- 支持通货、最低/最高价格、在线状态、标价类型、抓取条数和检查间隔筛选。
- 支持桌面提醒、复制第一条私聊文本和打开官方集市搜索页。
- 支持保存多个项目、勾选批量启动以及一行一个链接的快速导入。
- 不自动私聊、购买或绕过官方规则，不保存账号密码。
- `market-monitor-web-cn-v1.7/` 保留独立网页测试版和国服点击助手，便于单独调试。

## 累计更新记录

### 正则生成器

- 按简体中文编年史“引路石 Mods /111”维护 111 条词条/T 级快捷库。
- 带数值区间的词条自动生成输入框并填入 T 级范围；固定词条不生成数值输入。
- 支持游戏复制文本中的实际值、来源区间和反向区间格式。
- 修复自建预设保存时丢失 `must`、`any`、`exclude`、数值条件和字符限制的问题。
- 兼容旧的 `{ payload: {...} }` 预设格式。

### 物品导入与严格做装

- 采用精确底材池、词缀方向、T 级、中文语义和数值区间联合识别复制词缀。
- 支持品质、插槽、腐化、镜像、亵渎来源、多行词缀、符文与附魔注记。
- 单步状态机覆盖基础通货、骨材、精华、符文、灵魂核心、合金、增幅器和预兆。
- 切换预兆时保留当前物品、词缀、插槽和做装状态。
- 严格数据无法确认结果时阻止执行，不伪造概率或词缀映射。
- 修复部分预兆 `triggerCurrency` 为字符串时的页面崩溃。

### 中文数据与通货目录

- 具体物品名称与说明优先读取 `poe2db.tw/cn` 中文缓存。
- 改进 BaseType 表格解析，兼容拆分单元格、压缩单行和较长物品说明。
- 未可靠匹配的专有名称保留来源英文，不使用猜测翻译。
- 通货目录统一收录基础通货、亵渎材料、精华、符文、灵魂核心、插槽强化物和预兆。
- 支持分类以及中文、英文、作用和限制搜索，并标记当前底材可用性。

### 数据更新与运行安全

- 严格数据整组来自同一提交，完整下载、标准化和二次校验后才切换当前快照。
- 快照安装到 Electron `userData` 的版本化目录；失败时保留并恢复上一份完整快照。
- 下载使用随机 `.part` 文件和原子改名，对 Windows 文件占用做有限重试。
- Electron 缓存 SHA-256 不一致时重新下载并校验，失败文件不会解压或执行。
- 兑换助手使用单文件入口并记录运行诊断，避免控制器与启动脚本加载顺序不一致。

## 数据职责

- **严格数据图**：精确底材关系、词缀、T 级、冲突组和社区推导权重。
- **编年史中文层**：具体物品中文名称、说明和来源页面。
- **本地状态机规则**：通货、预兆、精华、骨材、合金和增幅器执行逻辑。
- **市场监控层**：官方搜索链接解析、区服 endpoint、结果过滤、串行轮询与本地提醒。

中文文本不参与概率计算；数据缺失时不会通过机翻或猜测补齐概率。

## 更新数据

进入做装模拟器后点击：

```text
更新严格数据与编年史中文
```

更新器会下载同一严格数据提交，重建精确底材池，同步中文缓存，校验后再原子切换。更新失败不会覆盖上一份可用快照。
