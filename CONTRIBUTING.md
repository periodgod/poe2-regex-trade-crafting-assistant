# 贡献指南

## 开发原则

1. 不把推测的词缀权重标记为官方数据。
2. 新增通货必须同时提供数据定义、状态机实现和测试。
3. 修改物品状态时必须经过 `validateItemState`。
4. 不新增账号登录、Cookie 持久化或内嵌远程交易网页。
5. 新功能必须通过 `npm run check`。

## 本地检查

```bash
npm run data:check
npm run check
```

## 数据贡献

- 底材放入 `data-v2/bases/`；
- 普通、精华、亵渎、腐化和固有词缀分别放入对应目录；
- 通货规则放入 `data-v2/currencies/`；
- 预兆规则放入 `data-v2/omens/`；
- 每条可验证规则应记录来源和核对日期；
- 不确定的权重必须标记低可信度，不能伪造精确概率。

## Pull Request

PR 中请说明：

- 修改的模块；
- 数据来源；
- 复现步骤；
- 新增或修改的测试；
- 是否影响既有存档或数据结构。
